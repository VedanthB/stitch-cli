Built for developers

Everything you need to integrate AI-generated UI into your workflow. Nothing you don't.

Zero Dependencies

Only Node.js built-in fetch(). No SDK, no framework, no bloat. Install and run instantly.

Pipe-Friendly

Stdout-first design. Pipe HTML to files, jq for JSON, or chain with other tools. Unix philosophy.

OAuth Login

One command to sign in with Google. Opens your browser, handles the PKCE flow, saves credentials. Run stitch login and go.

Real HTML Output

Download production-ready HTML/CSS directly. No intermediate formats, no conversion steps. Open in any browser.

MCP Protocol

Speaks Streamable HTTP (JSON-RPC) directly to Google's official MCP server. No proxy, no middleware.

Full Design Workflow

Create projects, generate screens, edit designs, produce variants, and download — all from one CLI.

Every command you need

Simple verbs, powerful results.

Command Description
stitch login Sign in with Google (browser-based OAuth)
stitch projects List all your projects
stitch screens <id> List screens in a project
stitch generate <id> "prompt" Generate a new screen from text
stitch edit <id> <screens> "prompt" Edit existing screens with AI
stitch download <id> <screen> Download screen as HTML file
stitch logout Remove saved credentials
--device

How it works

Three lines. That's the whole architecture.

1

Authenticate

stitch login opens your browser, runs OAuth2 with PKCE, and saves a refresh token locally. Or use an API key — your choice.

2

Talk MCP

The CLI sends JSON-RPC messages to stitch.googleapis.com/mcp using fetch(). Initialize session, call tools, get results. ~80 lines of client code.

3

Get HTML

Stitch's AI generates full-page designs. The CLI extracts the HTML download URL, fetches the content, and writes it to your file system. Production-ready.

Get started in seconds

One install. One login. Then generate.

# Install globally
$ npm install -g stitch-terminal
# Sign in
$ stitch login
# Generate a UI
$ stitch generate <project-id> "A dashboard with charts"
# Download the HTML
$ stitch download <project-id> <screen-id> -o dashboard.html
